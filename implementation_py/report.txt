Terminology quick note
- IID (in-distribution): eval data follows the same pattern family as training.
- OOD (out-of-distribution): eval data uses deliberately held-out pattern families.
- Visibility signature (v1): per-move seen/unseen bit pattern used to define holdout groups.
- OOD signature seed: seed that fixes which signatures are held out, independent of training seed.

Planned status delta (2026-05-03)
- New implementation track is approved in `tiny_lm/v1_0/`; `tiny_lm/v1/` remains unchanged.
- `v1_0` scope is intentionally narrow: sender receives a factual statement, receiver receives a question, receiver predicts the target location/container class.
- `v1_0` data plan: synthetic templated text pairs with hand-rolled shared word tokenization (`<PAD>`, `<UNK>`, `<BOS>`, `<EOS>`), with vocabulary built from train split sender+receiver text.
- `v1_0` model plan: sender encoder -> latent message -> receiver encoder/classifier, with ablations `with_message`, `zero_message`, and `shuffled_message`.
- `v1_0` reporting plan: track `with_message_acc`, `zero_message_acc`, `shuffled_message_acc`, and `comm_gain`; render class predictions into simple natural-language answers for demos.
- False-belief reasoning is explicitly deferred to `v1_1`; `v1_0` is communication-capacity first.

What "character-level" means (v0)
- In v0, each individual character is a token (letters, spaces, punctuation, newline).
- Example conceptually: "Hello" is modeled as H, e, l, l, o instead of whole words/subwords.
- Pros:
  - Very simple tokenizer and data pipeline.
  - No unknown word issue (any seen character can be generated).
  - Good for debugging model/training mechanics.
- Cons:
  - Long effective sequence lengths for the same content.
  - Weaker semantic abstraction than subword/word tokenization.
  - More prone to local repetition and less coherent long-form generation in tiny models.

Why this design was chosen first
- v0 is intended as a minimal, understandable baseline.
- Character-level keeps the stack small while validating core training, generation, tests, and checkpoint flow.

What we will do in the future

Near-term (v0 hardening)
- Keep v0 character-level for now.
- Improve reproducibility and diagnostics (run metadata, consistent eval prompts, better checkpoint logging).
- Keep v0 stable as a baseline reference; only apply targeted bug fixes and quality-of-life improvements.

Mid-term (v1 latent communication)
- Expand communication experiments with stronger evaluations:
  - ablations (`with_message`, `zero_message`, `shuffled_message`)
  - OOD splits and multiple seeds
  - message diagnostics (norm/variance, collapse checks)
  - probes for whether message encodes belief state vs shortcuts.
- Add qualitative latent-analysis reports tied to canonical ToM-style cases.

Long-term (possible extensions)
- Add alternative tokenization (subword/BPE) once baseline comparisons are stable.
- Explore discrete communication channels (e.g., VQ/Gumbel) for interpretability.
- Move from synthetic tasks toward richer environments while preserving ablation rigor.
